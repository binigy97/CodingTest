## 552 : 반복제어문3 - 자가진단5
### 문제
```
자연수 n을 입력받아서 다음과 같이 출력하는 프로그램을 작성하시오.
```
### 입력 예
```
3
```
### 출력 예
```
*****
 ***
  *
```

---

### 풀이
```java
import java.util.Scanner;

public class Main {
 public static void main(String[] args) {    
	 Scanner sc = new Scanner(System.in);
	 int n = sc.nextInt();
	 int temp = n * 2 - 1;
	 
	 for(int i = 0; i < n; i++) {
		 System.out.print(" ".repeat(i) + "*".repeat(temp) + " ".repeat(i));
		 System.out.println();
		 temp -= 2;
	 }
	 sc.close();
 }
}
```
2022.01 ~ 2022.05 자바의 제출현황을 살펴보면 정답인 결과 중의 실행 시간은 다음과 같다.
- 2022.01 : 400 ~ 600ms
- 2022.02 ~ 2022.05 : 250ms ~ 350ms
- 위의 코드 : 194ms

#### :question: 내 코드가 빠른 이유는?
[정올 634번](https://github.com/binigy97/CodingTest/blob/main/Jungol/loop3/634.md)에서 이중 반복문과 단일 반복문에 repeat() 함수를 사용한 코드 간의 실행 시간을 비교해 보았지만 
두 코드는 유의미한 차이가 없었다. 그러나 이번엔, 유의미한 결과가 나타났다.  

해당 문제를 구글링해 보면 자바 코드가 거의 다 똑같다. 그 코드를 복사하여 실행하니 379ms 걸렸다(두 코드 모두 반복적인 테스트는 진행하지 않음). 
내 코드는 약 200ms, 남의 코드는 300ms...? 왜 그런지 생각해 봤다.  

**다중 for문의 경우 외부보다 내부 반복문의 반복횟수가 많아질수록 실행 시간이 오래 소요된다고 한다.** 
repeat() 함수도 살펴보면 for문으로 이루어져 있으므로 repeat()를 사용한 위 코드도 다중 반복문이라고 볼 수 있다. 차이가 있다면 repeat()는 class 파일을 실행시킨다는 것..? 
어찌보면 더 많은 연산 과정이 정의된 repeat()가 더 복잡한 코드임에도 **속도 차이가 발생한 이유는, repeat()를 사용함으로써 내부 반복문은 class 파일을 바로 이용했기 때문이라고 볼 수 있을 것 같다.** ~~이게 맞았으면 좋겠다..ㅎㅎ~~  

참고로 위 문제의 C는 3ms 걸린다. C는 assembly 다음으로 빠른 언어이니 Java와 비교했을 때 60 ~ 200배의 차이가 발생하고 있다. 
안 되는 건 알지만, Java로 C에 필적할 만한 코드를 짜보고 싶다. 위 문제를 100ms 이하로 낮출 수 있는 코드는 없을까..?
